## Voice-to-Image AI Research

Этот репозиторий предназначен для магистерского диплома по теме:

**«Семантическое структурирование и обогащение речевых описаний для генерации изображений средствами нейронных сетей»**

**Фокус работы**: преобразование естественной человеческой речи в структурированные и обогащённые текстовые промпты для генеративных моделей изображений (Stable Diffusion, SDXL, Flux и др.).

**Основные компоненты**:
- распознавание речи и учёт личности говорящего;
- семантическое структурирование речевых описаний;
- обогащение промптов (добавление деталей, стилистических маркеров, технических тегов);
- генерация изображений с учётом текста и референсных картинок;
- быстрая работа на ноутбуках без мощных GPU и на серверах с GPU;
- архитектура в виде нескольких микросервисов и воспроизводимые эксперименты.

**Подробнее о теме диплома**: см. [`docs/thesis_topic.md`](docs/thesis_topic.md)

### Основные компоненты

- **ASR (Automatic Speech Recognition)**: локальное распознавание речи (Whisper / faster-whisper, Vosk).
- **Speaker ID / пресеты пользователей**: эмбеддинги голоса и мэппинг пользователя → стили и пресеты.
- **Text / Image → Image генерация**: Stable Diffusion / Turbo / LCM через `diffusers` + возможный экспорт в ONNX.
- **MLOps / эксперименты**: `MLflow` для трекинга экспериментов, моделей и конфигураций.
- **API / микросервисы**: `FastAPI` + Docker (в перспективе) для разделения на сервисы ASR, генерации и оркестрации.

### Структура репозитория (черновой план)

- `notebooks/`
  - `01_asr_and_speaker_id.ipynb` — распознавание речи, базовая диаризация / спикер-ID, связывание пользователя с пресетами.
  - `02_text_and_image_conditioning.ipynb` — подготовка промптов и работа с референсными изображениями.
  - `03_fast_image_generation_and_benchmark.ipynb` — эксперименты по ускорению генерации на CPU/GPU.
  - `04_online_meeting_demo.ipynb` — прототип онлайн-встречи: несколько голосов → динамически меняющееся изображение.
- `docs/`
  - `thesis_topic.md` — формулировка темы диплома, цели, задачи, структура работы.
  - `libraries_comparison.md` — сравнение библиотек и фреймворков.
  - `architecture.md` — архитектура микросервисов и сценариев.
- `services/` (позже)
  - `asr_service/`, `image_service/`, `orchestrator_service/` — каркасы FastAPI-сервисов.

### Как запускать ноутбуки

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Запустить Jupyter:

```bash
jupyter lab
```

или

```bash
jupyter notebook
```

3. Открыть нужный ноутбук из папки `notebooks/` и следовать шагам внутри.
